---

## Overview

ciao.zinnias is a mobile-first web service for small communities (10–100 members) to share schedules, confirm attendance, and leave short notes. It is designed for people who are comfortable using smartphones but who dislike passwords, accounts, and complex apps.

Members join with a one-time invite code. They check upcoming events, tap Going or No Go, and optionally add a short note. Admins create events, generate invite codes, and manage the member list. There is no registration form, no password, and no social feed.

---

## Why / When

Use ciao.zinnias when a community needs a **private shared calendar with attendance** and the alternatives are either too heavy (full groupware) or too informal (a chat thread where information gets buried).

It is a good fit for:

- Neighbourhood groups, volunteer teams, hobby circles
- Communities where some members are non-technical
- Cases where a shared LINE/WhatsApp group is used today but schedules get lost

It is not a fit for large organisations, public event listings, real-time chat, or anything requiring fine-grained permissions beyond admin and member.

## Quick Start

**Prerequisites:** Rust stable, `wasm32-unknown-unknown` target, Bun (or Node.js ≥ 18).

```sh
# One-time setup: install wrangler, apply migrations, seed dev data
bun run setup

# Start the local dev server on http://localhost:8787
bun run dev

# Run the test suite
cargo test -p zinnias-ciao-domain -p zinnias-ciao-contracts

# Type-check the SSR worker for wasm32 (required before every commit)
cargo check -p zinnias-ciao-ssr --target wasm32-unknown-unknown
```

The setup script prints an invite code. Open `http://localhost:8787/join` and paste it to join the seeded community.

Full setup and deployment instructions are in [`docs/src/quick-start.md`](docs/src/quick-start.md) and [`docs/src/deployment.md`](docs/src/deployment.md).

## Design Notes

**Pure server-side rendering.** The app works without JavaScript. Forms use `<form method="post">` with 303 redirects. Small progressive enhancements (offline banner, note character counter) are layered on top but are never required for core functionality.

**Invite-only, no passwords.** Members join with a one-time code generated by an admin. Sessions are HMAC-secured cookies. No email, no OAuth, no password reset flow in the current release.

**Offline-aware.** The service worker caches previously visited pages for read-only offline use. The app shows a clear banner when offline. Form submissions require connectivity (the browser handles the network error directly).

**Recurring events.** Admins can create weekly, bi-weekly, or monthly repeating events. Each occurrence is a concrete event with its own attendance — no abstract recurrence rules visible to members.

**Templates.** Common event details (title, location, default duration) can be saved as templates and used to pre-fill the create-event form.

**Community data export.** Admins can download all community data as a JSON file (events, attendance, member names, notes). Session tokens and invite secrets are never included.
